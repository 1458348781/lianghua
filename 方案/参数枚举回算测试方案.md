# 涨停回落策略参数枚举回算测试方案

## 1. 测试目标

对当前涨停回落/次日确认策略做参数枚举回算，分别统计 2024、2025、2026 三个年度的表现，找出：

1. 单年度收益、回撤、胜率、交易次数较稳的参数组合。
2. 跨年度都有效、不是只在单一年份偶然赚钱的参数组合。
3. 对策略结果影响最大的参数区间，后续用于收窄实盘观察参数。

2026 年建议按当前可用数据回算：`2026-01-01` 至 `2026-05-09`，后续有新数据后可滚动补算。

## 2. 当前策略关键参数

当前核心参数在 `quant-platform/scripts/limit_up_pullback_strategy.py` 的 `BacktestConfig` 中，主要分为五类：

### 2.1 股票池过滤参数

| 参数 | 当前值 | 作用 |
|---|---:|---|
| `min_list_days` | 250 | 排除上市时间太短的新股 |
| `min_float_mv` | 20 亿 | 最小流通市值，BaoStock 版本暂不可用 |
| `max_float_mv` | 200 亿 | 最大流通市值，BaoStock 版本暂不可用 |
| `min_price` | 3 | 最低股价 |
| `max_price` | 500| 最高股价 |

### 2.2 涨停日 T 日过滤参数

| 参数 | 当前值 | 作用 |
|---|---:|---|
| `min_turnover` | 3.0 | 涨停日最小换手率 |
| `max_turnover` | 25.0 | 涨停日最大换手率 |
| `day1_min_volume_ratio` | 1.2 | 涨停日成交量 / 前 5 日均量下限 |
| `day1_max_volume_ratio` | 6.0 | 涨停日成交量 / 前 5 日均量上限 |
| `range_min_amplitude_30` | 0.18 | 前 30 日振幅下限 |
| `range_min_return_20` | 0.05 | 前 20 日涨幅下限 |

### 2.3 T+1 确认日过滤参数

| 参数 | 当前值 | 作用 |
|---|---:|---|
| `day2_min_pct_chg` | -3.0 | T+1 涨跌幅下限 |
| `day2_max_pct_chg` | 8.0 | T+1 涨跌幅上限 |
| 内置条件 | 1.8 | T+1 成交量不超过 T 日成交量 1.8 倍 |
| 内置条件 | 0.45 | T+1 收盘位置不低于全天区间 45% |
| 内置条件 | 0.06 | T+1 上影线比例不高于 6% |
| 内置条件 | 0.97 | T+1 收盘价不低于 T 日收盘价 97% |

### 2.4 T+2 入场过滤参数

| 参数 | 当前值 | 作用 |
|---|---:|---|
| `entry_min_open_gap_pct_chg` | 1.0 | T+2 开盘高开下限 |
| `entry_max_open_gap_pct_chg` | 5.0 | T+2 开盘高开上限 |
| `entry_min_high_from_open_pct_chg` | 3.0 | T+2 盘中相对开盘最大涨幅下限 |

### 2.5 持仓和风控参数

| 参数 | 当前值 | 作用 |
|---|---:|---|
| `max_positions` | 5 | 最大持仓数量 |
| `hold_days` | 3 | 当前脚本参数中有，但核心卖出逻辑暂未使用到期退出 |
| `stop_loss` | -0.05 | 止损线 |
| `take_profit` | 0.16 | 当前脚本参数中有，但核心卖出逻辑暂未使用固定止盈 |
| `strong_close_pct_chg` | 3.0 | 持仓日收盘强度，不满足则弱势卖出 |

## 3. 枚举原则

不要一开始做全量笛卡尔积暴力枚举，否则组合数会爆炸，而且容易过拟合。建议分三层：

1. **第一层：粗网格筛选**  
   控制在 500～1500 组参数以内，先找出明显有效区间。

2. **第二层：重点参数细化**  
   只围绕第一层排名靠前且交易次数足够的组合，细化 3～5 个最敏感参数。

3. **第三层：稳健性复核**  
   用 2024、2025、2026 分年度分别排名，并增加跨年综合评分，避免只选出某一年行情适配的组合。

## 4. 建议第一轮粗网格枚举范围

第一轮建议优先枚举对信号数量和交易质量影响最大的参数，暂时固定交易成本、初始资金、股票池基础过滤。

### 4.1 固定参数

| 参数 | 固定值 | 说明 |
|---|---:|---|
| `initial_cash` | 1,000,000 | 保持不同组合可比 |
| `commission_rate` | 0.0003 | 当前默认手续费 |
| `slippage_rate` | 0.0005 | 当前默认滑点 |
| `stamp_tax_rate` | 0.001 | 当前默认印花税 |
| `min_list_days` | 250 | 先固定，避免股票池变化太大 |
| `min_price` | 3 | 先固定 |
| `max_price` | 80 | 先固定 |
| `max_positions` | 5 | 先固定持仓容量 |

### 4.2 第一轮粗网格

| 参数 | 建议枚举值 | 组合数 |
|---|---|---:|
| `min_turnover` | `[2.0, 3.0, 5.0]` | 3 |
| `max_turnover` | `[20.0, 25.0, 35.0]` | 3 |
| `day1_min_volume_ratio` | `[1.0, 1.2, 1.5]` | 3 |
| `day1_max_volume_ratio` | `[4.0, 6.0, 8.0]` | 3 |
| `range_min_amplitude_30` | `[0.12, 0.18, 0.25]` | 3 |
| `range_min_return_20` | `[0.00, 0.05, 0.10]` | 3 |
| `day2_min_pct_chg` | `[-5.0, -3.0, 0.0]` | 3 |
| `day2_max_pct_chg` | `[5.0, 8.0, 12.0]` | 3 |
| `entry_min_open_gap_pct_chg` | `[0.0, 1.0, 2.0]` | 3 |
| `entry_max_open_gap_pct_chg` | `[4.0, 5.0, 7.0]` | 3 |
| `entry_min_high_from_open_pct_chg` | `[2.0, 3.0, 4.0]` | 3 |
| `stop_loss` | `[-0.03, -0.05, -0.08]` | 3 |
| `strong_close_pct_chg` | `[1.0, 3.0, 5.0]` | 3 |

理论组合数是 `3^13 = 1,594,323`，不建议全跑。第一轮应采用**分组枚举**，而不是所有参数同时笛卡尔积。

## 5. 推荐实际枚举批次

### 批次 A：信号数量参数

目标：先找到交易次数足够、不是过严也不是过松的信号区间。

固定其他参数为默认值，只枚举：

| 参数 | 枚举值 |
|---|---|
| `min_turnover` | `[2.0, 3.0, 5.0]` |
| `max_turnover` | `[20.0, 25.0, 35.0]` |
| `day1_min_volume_ratio` | `[1.0, 1.2, 1.5]` |
| `day1_max_volume_ratio` | `[4.0, 6.0, 8.0]` |
| `range_min_amplitude_30` | `[0.12, 0.18, 0.25]` |
| `range_min_return_20` | `[0.00, 0.05, 0.10]` |

组合数：`3^6 = 729`。

筛选条件：

- 2024、2025、2026 合计买入次数不少于 20 次。
- 任一年度买入次数不为 0。
- 最大回撤不能明显失控，建议先卡在 `>-25%`。

### 批次 B：确认日和入场参数

目标：在批次 A 的前 20～50 组基础上，优化 T+1/T+2 的确认强度。

对批次 A 选出的候选组合继续枚举：

| 参数 | 枚举值 |
|---|---|
| `day2_min_pct_chg` | `[-5.0, -3.0, 0.0]` |
| `day2_max_pct_chg` | `[5.0, 8.0, 12.0]` |
| `entry_min_open_gap_pct_chg` | `[0.0, 1.0, 2.0]` |
| `entry_max_open_gap_pct_chg` | `[4.0, 5.0, 7.0]` |
| `entry_min_high_from_open_pct_chg` | `[2.0, 3.0, 4.0]` |

单个候选组合扩展数：`3^5 = 243`。如果保留 30 个候选，则约 7290 组，可以接受。

筛选重点：

- 交易次数不要因为条件过严而骤降。
- 优先选择 2024、2025、2026 都有交易的组合。
- 优先选择夏普、收益回撤比稳定的组合，不只看累计收益。

### 批次 C：风控参数

目标：在批次 B 的前 20～30 组基础上，调整止损和弱势卖出强度。

| 参数 | 枚举值 |
|---|---|
| `stop_loss` | `[-0.03, -0.05, -0.08]` |
| `strong_close_pct_chg` | `[1.0, 3.0, 5.0]` |
| `max_positions` | `[3, 5, 8]` |

单个候选组合扩展数：`3^3 = 27`。

说明：

- `stop_loss` 越紧，回撤可能变小，但容易被日内波动洗出。
- `strong_close_pct_chg` 越高，卖出越严格，持仓更短，可能提升资金周转，也可能错过后续上涨。
- `max_positions` 影响仓位分散程度和单票收益贡献，建议最后再测。

## 6. 不建议第一轮枚举的参数

以下参数建议先不动，等核心参数稳定后再测：

| 参数 | 原因 |
|---|---|
| `commission_rate` / `slippage_rate` / `stamp_tax_rate` | 这是交易成本假设，先固定，否则会混淆策略参数效果 |
| `initial_cash` | 只影响规模，不应影响百分比表现，除非交易金额太小导致整手约束明显 |
| `min_float_mv` / `max_float_mv` | BaoStock 版本暂没有流通市值，当前不适合纳入主枚举 |
| `take_profit` | 当前核心卖出逻辑暂未使用固定止盈，先不要枚举 |
| `hold_days` | 当前核心卖出逻辑暂未使用到期退出，先不要枚举；如果要测，应先补实现 |

## 7. 回算年度设置

建议每个参数组合都跑三个年度独立结果：

| 年份 | 起始日期 | 结束日期 | 备注 |
|---|---|---|---|
| 2024 | `2024-01-01` | `2024-12-31` | 完整年度 |
| 2025 | `2025-01-01` | `2025-12-31` | 完整年度，需确认数据已下载 |
| 2026 | `2026-01-01` | `2026-05-09` | 当前可用年度，后续滚动更新 |

每个年度需要保留至少 80～100 个自然日 warmup 数据，用于计算 20 日收益、30 日振幅和 5 日均量。实际拉取数据时，起始数据日期应早于回测起始日期，例如：

- 2024 回算数据从 `2023-09-23` 左右开始取。
- 2025 回算数据从 `2024-09-23` 左右开始取。
- 2026 回算数据从 `2025-09-23` 左右开始取。

## 8. 输出结果文件建议

建议在当前项目下新增目录：

```text
quant-platform/reports/parameter_sweep/
```

每次枚举生成以下文件：

```text
parameter_sweep_summary_YYYYMMDD_HHMMSS.csv
parameter_sweep_top_YYYYMMDD_HHMMSS.csv
parameter_sweep_config_YYYYMMDD_HHMMSS.json
```

## 9. 排名评分建议

不要只按收益排序，建议使用综合评分：

```text
score = mean_annual_return
        + 0.5 * min_annual_return
        + 0.1 * mean_sharpe
        - 0.8 * abs(worst_max_drawdown)
        - trade_penalty
```

其中：

```text
trade_penalty = 0.2 if sum_buy_count < 20 else 0
```

筛选优先级：

1. `sum_buy_count >= 20`。
2. `min_year_buy_count > 0`。
3. `worst_max_drawdown > -0.25`。
4. `min_annual_return` 不要明显为大负数。
5. 再看 `score`、`mean_sharpe`、`mean_annual_return`。

## 10. 建议最终保留的结果

最终不要只选 1 组最优参数，建议保留三类参数：

1. **收益型参数**：平均收益最高，但允许回撤略大。
2. **稳健型参数**：三年表现都较均衡，最差年度不太差。
3. **低回撤型参数**：收益不是最高，但最大回撤最小。

每类保留 3～5 组，后续再用更长历史区间或实盘模拟观察。

## 11. 执行顺序建议

1. 先补一个 `parameter_sweep.py` 脚本，复用现有 `build_signals`、`run_backtest`、`summarize`。
2. 优先使用本地缓存数据，避免每组参数重复下载行情。
3. 先跑批次 A，生成粗筛结果。
4. 按筛选规则选出前 20～50 组。
5. 跑批次 B，细化确认日和入场参数。
6. 跑批次 C，细化风控和持仓数量。
7. 输出跨年度排名表和每年明细表。
8. 人工复核前 10～20 组参数的交易明细，确认不是单票偶然贡献。

## 12. 当前建议优先测试的默认候选

如果希望先快速跑一小批，可以从下面 8 组开始：

| 组别 | 特征 | 调整方向 |
|---|---|---|
| baseline | 当前默认参数 | 作为基准 |
| loose_volume | 放宽量能 | `day1_min_volume_ratio=1.0`, `day1_max_volume_ratio=8.0` |
| strict_volume | 收紧量能 | `day1_min_volume_ratio=1.5`, `day1_max_volume_ratio=4.0` |
| loose_entry | 放宽入场 | `entry_min_open_gap_pct_chg=0.0`, `entry_min_high_from_open_pct_chg=2.0` |
| strict_entry | 收紧入场 | `entry_min_open_gap_pct_chg=2.0`, `entry_min_high_from_open_pct_chg=4.0` |
| loose_confirm | 放宽 T+1 | `day2_min_pct_chg=-5.0`, `day2_max_pct_chg=12.0` |
| strict_confirm | 收紧 T+1 | `day2_min_pct_chg=0.0`, `day2_max_pct_chg=5.0` |
| defensive_exit | 防守卖出 | `stop_loss=-0.03`, `strong_close_pct_chg=5.0` |

这 8 组适合先确认 2024、2025、2026 是否都有足够信号，再决定是否进入大规模枚举。
