# 分歧流2策略参数优化回测方案

> 目标：基于 2020、2021、2022、2023、2024、2025、2026 年历史数据，对“分歧流2”策略参数进行系统回测和优化，找到在收益、回撤、稳定性、交易次数之间综合表现最优的一组或多组参数。

---

## 一、策略目标

本方案不是单纯寻找历史收益最高的参数，而是寻找“可实盘使用”的稳健参数。

优化目标应同时考虑：

1. 年化收益率；
2. 最大回撤；
3. 胜率；
4. 盈亏比；
5. 夏普比率；
6. Calmar 比率；
7. 交易次数；
8. 每年表现是否稳定；
9. 是否过度依赖某一年行情；
10. 参数是否对小幅变化不敏感。

最终输出不只是一组“历史最优参数”，而应输出：

- 全样本最优参数；
- 分年度最优参数；
- 稳健参数区间；
- 样本内/样本外验证结果；
- 推荐实盘参数；
- 备用保守参数。

### 1. 计算参数

| 参数 | 当前值 | 含义 |
|---|---:|---|

| hold_days | 3 | 持仓天数 |

| min_price | 3 | 最低股价 |
| max_price | 500 | 最高股价 |
| min_turnover | 2 | T 日换手率下限 2% |
| max_turnover | 25 | T 日换手率上限 25% |
| day1_min_volume_ratio | 1 | T 日成交量 >= 过去 5 日均量 * 1 |
| day1_max_volume_ratio | 4 | T 日成交量 <= 过去 5 日均量 * 4 |
| range_min_amplitude_30 | 0.18 | 过去 30 日振幅 >= 18% |
| range_min_return_20 | 0 | 过去 20 日涨幅 >= 0% |
| day2_min_pct_chg | -3 | T+1 涨跌幅下限 -3% |
| day2_max_pct_chg | 8 | T+1 涨跌幅上限 8% |
| entry_min_open_gap_pct_chg | 1 | T+2 高开下限 1% |
| entry_max_open_gap_pct_chg | 5 | T+2 高开上限 5% |
| entry_min_high_from_open_pct_chg | 4 | T+2 盘中最高价较开盘至少涨 4% |

### 2. 固定条件

| 条件 | 含义 |
|---|---|
| T 日必须涨停 | T 日为情绪启动日 |
| T 日不能是一字板 | 必须有真实换手 |
| T 日不能是 ST | 排除风险股 |
| T+1 日不能是 ST | 排除状态变化风险 |
| T+1 成交量 <= T 日成交量 * 1.8 | 防止分歧放量出货 |
| T+1 收盘位置 >= 0.45 | 收盘不能太弱 |
| T+1 上影线 <= 6% | 防止冲高回落过强 |
| T+1 收盘价 >= T 日收盘价 * 0.97 | 不允许明显跌破 T 日涨停价 |
| T+2 最高价必须突破 T+1 最高价 | 必须继续走强 |

### 3. 买点规则

T+2 买入必须同时满足：

1. T+2 开盘涨幅在 1% ~ 5%；
2. T+2 盘中最高价较开盘涨幅 >= 4%；
3. T+2 最高价突破 T+1 最高价。

买入价：

```text
buy_price = max(T+1最高价, T+2开盘价 * 1.04)
```

## 四、参数优化总体思路

建议使用“三阶段优化法”：

1. **粗筛阶段**：用较大的步长快速找到有效区域；
2. **精筛阶段**：在有效区域内用更小步长精细搜索；
3. **稳健性验证阶段**：用滚动样本外和扰动测试验证参数是否可靠。

这样比直接全量暴力枚举更快，也更不容易得到过拟合结果。

# 五、推荐优化方案

## 方案 A：分阶段网格搜索，作为保底方案

这是最稳、最容易实现、最容易复现的方案。

### 1. 第一阶段：粗粒度枚举

先用较大步长搜索大范围参数，找到表现较好的区域。

建议范围：

| 参数 | 搜索范围 | 步长 |
|---|---:|---:|
| max_positions | 1 ~ 6 | 1 |
| hold_days | 1 ~ 8 | 1 |
| stop_loss | -0.08 ~ -0.01 | 0.01 |
| strong_close_pct_chg | 3 ~ 9 | 1 |
| min_price | 2 ~ 8 | 1 |
| max_price | 100 ~ 500 | 50 |
| min_turnover | 1 ~ 8 | 1 |
| max_turnover | 12 ~ 35 | 2 |
| day1_min_volume_ratio | 0.8 ~ 2.0 | 0.2 |
| day1_max_volume_ratio | 2.0 ~ 6.0 | 0.5 |
| range_min_amplitude_30 | 0.10 ~ 0.35 | 0.05 |
| range_min_return_20 | -0.10 ~ 0.20 | 0.05 |
| day2_min_pct_chg | -6 ~ 0 | 1 |
| day2_max_pct_chg | 5 ~ 9 | 1 |
| entry_min_open_gap_pct_chg | 0 ~ 3 | 0.5 |
| entry_max_open_gap_pct_chg | 3 ~ 8 | 0.5 |
| entry_min_high_from_open_pct_chg | 2 ~ 7 | 0.5 |

第一阶段不追求极致，只筛出前 1%—5% 的参数组合。

### 2. 第二阶段：细粒度搜索

围绕第一阶段表现最好的参数区域，缩小范围并降低步长。

建议步长：

| 参数类型 | 精细步长 |
|---|---:|
| 涨跌幅类参数 | 0.1% ~ 0.25% |
| 换手率类参数 | 0.1% ~ 0.5% |
| 成交量倍数 | 0.05 ~ 0.1 |
| 振幅/涨幅比例 | 0.005 ~ 0.01 |
| 持仓天数 | 1 天 |
| 最大持仓数 | 1 个 |

用户要求“每个参数最多 0.1 的变化，可以更低”，建议在最终精筛阶段采用：

- 涨跌幅参数：0.1；
- 换手率参数：0.1；
- 成交量倍数：0.05；
- 振幅参数：0.005；
- 收益率参数：0.005。

### 3. 第三阶段：局部扰动测试

对最终候选参数做小幅扰动。

例如：

```text
entry_min_open_gap_pct_chg = 1.0
测试：0.8、0.9、1.0、1.1、1.2
```

如果最优参数附近的多个组合表现都较好，说明参数稳健。

如果只有一个点表现极好，周边参数全部变差，说明大概率过拟合。

## 方案 B：贝叶斯优化，推荐作为主方案

如果希望“又快又好”，推荐使用贝叶斯优化，而不是直接全参数暴力枚举。

推荐工具：

```text
Optuna
```

原因：

1. 实现简单；
2. 支持连续参数、整数参数、条件参数；
3. 支持并行；
4. 可以自动剪枝；
5. 比随机搜索和网格搜索更快找到优质区域；
6. 适合这种参数较多、回测较耗时的问题。

### 1. 优化流程

1. 先固定不可变策略逻辑；
2. 定义参数搜索空间；
3. 每次 Optuna 生成一组参数；
4. 对 2020—2026 数据执行回测；
5. 返回综合评分；
6. Optuna 根据历史结果继续选择下一组参数；
7. 运行 2,000—20,000 次 trial；
8. 取综合评分前 100 组参数；
9. 做样本外验证和稳定性测试。

### 2. 推荐 trial 数量

| 阶段 | trial 数量 |
|---|---:|
| 快速验证 | 500 ~ 1,000 |
| 正式优化 | 5,000 ~ 20,000 |
| 深度优化 | 50,000+ |

如果单次回测已经向量化，建议至少跑 10,000 次。

### 3. 推荐评分函数

不要只用总收益作为目标函数。

推荐综合评分：

```text
score = 年化收益率 * 0.35
      + Calmar比率 * 0.25
      + Sharpe比率 * 0.15
      + 胜率 * 0.10
      + 盈亏比 * 0.10
      - 最大回撤惩罚 * 0.30
      - 年度不稳定惩罚 * 0.20
      - 交易次数过少惩罚 * 0.15
```

也可以采用更实用的排序逻辑：

1. 交易次数必须大于最低阈值；
2. 最大回撤必须低于上限；
3. 每年至少不能严重亏损；
4. 在满足以上条件后，按 Calmar 比率和总收益排序。

推荐硬性过滤条件：

| 指标 | 建议要求 |
|---|---:|
| 2020—2026 总交易次数 | >= 100 次 |
| 单年交易次数 | >= 10 次，2026 可适当降低 |
| 最大回撤 | <= 25% |
| 亏损年份数量 | <= 2 年 |
| 任一年亏损 | 不超过 -15% |
| 胜率 | >= 45% |
| 盈亏比 | >= 1.1 |

## 方案 C：遗传算法/进化搜索，作为备选方案

如果参数组合空间非常大，可以使用遗传算法。

流程：

1. 随机生成一批参数种群；
2. 回测每组参数；
3. 保留表现最好的前 10%—20%；
4. 对优秀参数进行交叉、变异；
5. 继续回测新一代参数；
6. 迭代 50—200 代；
7. 输出最优参数和稳定参数区间。

优点：

- 适合参数很多的情况；
- 可以跳出局部最优；
- 比全量枚举快。

缺点：

- 结果可解释性不如网格搜索；
- 需要控制随机种子；
- 不如 Optuna 简单稳定。

因此，建议优先使用：

```text
Optuna 贝叶斯优化 + 局部网格精筛
```

# 六、GPU 使用方案

这台电脑有 GPU，但是否能明显加速，取决于回测实现方式。

## 1. GPU 适合加速的部分

GPU 适合做：

1. 大量股票、多日期的条件过滤；
2. 参数矩阵批量计算；
3. 大规模向量化收益计算；
4. 多参数组合并行评估；
5. 使用 cuDF / CuPy / Numba CUDA 的数组计算。

## 2. GPU 不适合的部分

GPU 不太适合：

1. 逐笔循环；
2. 复杂 if-else 串行逻辑；
3. 频繁读写小文件；
4. 每次 trial 都重新加载数据；
5. Python for 循环主导的回测。

如果代码还是普通 pandas + for 循环，GPU 基本发挥不出来。

## 3. 推荐加速架构

### 首选架构

```text
数据预处理：Polars / pandas
核心回测：NumPy / Numba
参数优化：Optuna 多进程并行
大规模矩阵：可选 CuPy / RAPIDS cuDF
```

### Windows 环境建议

因为当前电脑是 Windows，RAPIDS cuDF 在 Windows 原生环境支持不如 Linux 方便。

推荐优先级：

1. **NumPy + Numba JIT**：最稳，Windows 直接可用；
2. **Optuna 多进程并行**：CPU 多核加速明显；
3. **CuPy**：如果 CUDA 环境正常，可用于矩阵批量计算；
4. **WSL2 + RAPIDS cuDF**：如果愿意配置 Linux 子系统，GPU 加速能力更强。

## 4. 推荐实际方案

最推荐的实际计算方案是：

```text
第一层：用 Polars/pandas 预处理所有固定字段
第二层：把行情数据转为 NumPy 数组
第三层：用 Numba 编译核心回测循环
第四层：用 Optuna 负责参数搜索
第五层：用多进程并行跑 trial
第六层：对前 100 组参数做局部网格精筛
```

这个方案通常比直接上 GPU 更容易稳定跑起来，而且速度已经足够快。

如果后续数据量非常大，再把“参数组合批量评估”迁移到 CuPy 或 Numba CUDA。

# 七、参数搜索空间设计

## 1. 建议优先优化的参数

这些参数对策略结果影响最大，应优先优化：

| 优先级 | 参数 | 原因 |
|---|---|---|
| 1 | day2_min_pct_chg | 控制分歧不能太弱 |
| 1 | day2_max_pct_chg | 控制分歧不能太强 |
| 1 | entry_min_open_gap_pct_chg | 影响 T+2 买点质量 |
| 1 | entry_max_open_gap_pct_chg | 影响追高风险 |
| 1 | entry_min_high_from_open_pct_chg | 影响确认强度 |
| 1 | stop_loss | 直接影响回撤 |
| 2 | T+1 成交量倍数 | 区分分歧和出货 |
| 2 | T+1 收盘位置 | 判断承接强弱 |
| 2 | T+1 上影线 | 过滤冲高回落 |
| 2 | hold_days | 影响收益兑现周期 |
| 3 | min_turnover / max_turnover | 影响流动性和弹性 |
| 3 | day1 volume ratio | 判断启动放量是否健康 |
| 3 | range_min_amplitude_30 | 判断是否有波动空间 |
| 3 | range_min_return_20 | 判断短期趋势 |
| 4 | min_price / max_price | 更多是风控过滤 |
| 4 | max_positions | 影响组合层面资金曲线 |

## 2. 建议纳入优化的固定条件

当前部分“固定参数”也建议变成可优化参数，因为它们会显著影响结果。

建议加入优化：

| 当前固定条件 | 建议搜索范围 |
|---|---:|
| T+1 成交量 <= T日成交量 * 1.8 | 1.2 ~ 2.2，步长 0.05 |
| T+1 收盘位置 >= 0.45 | 0.35 ~ 0.75，步长 0.05 |
| T+1 上影线 <= 6% | 3% ~ 10%，步长 0.5% |
| T+1 收盘价 >= T日收盘价 * 0.97 | 0.94 ~ 1.02，步长 0.005 |

不建议优化的固定条件：

- T 日必须涨停；
- T 日不能是一字板；
- T 日不能是 ST；
- T+1 日不能是 ST；
- T+2 最高价必须突破 T+1 最高价。

这些是策略逻辑核心，不建议放开。

# 八、完整回测流程

## 第一步：数据准备

1. 获取 2020—2026 全 A 日线数据；
2. 复权方式统一，建议使用前复权；
3. 剔除停牌、退市、异常数据；
4. 标记 ST 状态；
5. 标记涨停价和是否涨停；
6. 标记一字板；
7. 计算过去 5 日均量、过去 20 日涨幅、过去 30 日振幅；
8. 保存为 parquet 格式，避免反复读 CSV。

## 第二步：事件样本生成

先固定不变条件，生成所有候选事件。

候选事件以 T 日为基准，每条记录包含：

- T 日行情；
- T+1 行情；
- T+2 行情；
- 后续 N 日行情；
- 所需派生字段。

这样参数优化时不需要每次重新扫描全市场，只需要在候选事件表上做过滤。

这是提速的关键。

## 第三步：参数过滤

每组参数只在候选事件表上做布尔过滤：

1. 价格区间过滤；
2. 换手率过滤；
3. T 日成交量倍数过滤；
4. 30 日振幅过滤；
5. 20 日涨幅过滤；
6. T+1 涨跌幅过滤；
7. T+1 成交量过滤；
8. T+1 收盘位置过滤；
9. T+1 上影线过滤；
10. T+1 收盘价过滤；
11. T+2 高开过滤；
12. T+2 盘中强度过滤；
13. T+2 突破 T+1 高点过滤。

## 第四步：模拟买入

买入价为：

```text
buy_price = max(T+1 high, T+2 open * (1 + entry_min_high_from_open_pct_chg / 100))
```

需要注意：

- 如果买入价高于 T+2 最高价，则视为没有成交；
- 如果买入价等于或低于 T+2 最高价，则视为成交；
- 成交后用 buy_price 作为成本价。

## 第五步：模拟卖出

默认持仓：

```text
hold_days = 3
```

卖出规则建议：

1. 买入后任意一天最低价触发 stop_loss，则按止损价卖出；
2. 未触发止损，则持有到第 hold_days 天收盘卖出；
3. 如果期间停牌，按可交易日顺延；
4. 如果一字跌停无法卖出，需要单独处理滑点和不可成交风险。

止损价：

```text
stop_price = buy_price * (1 + stop_loss)
```

## 第六步：组合层面约束

因为 max_positions = 3，需要处理同一天多个信号的排序问题。

建议排序规则：

1. 优先选择所属最强板块标的；
2. 其次选择 T+1 收盘位置更高的；
3. 其次选择 T+2 高开幅度适中的；
4. 其次选择 T 日成交额更高的；
5. 最后按股票代码排序，保证回测可复现。

如果没有板块数据，建议排序：

```text
T+1收盘位置 desc
T+2最高突破强度 desc
T日成交额 desc
T+1成交量倍数 asc
```

同一天最多买入 max_positions 只股票。

## 第七步：费用和滑点

必须加入交易成本，否则结果会偏乐观。

建议默认：

| 项目 | 参数 |
|---|---:|
| 买入佣金 | 0.025% |
| 卖出佣金 | 0.025% |
| 印花税 | 0.05% 或按当前规则配置 |
| 单边滑点 | 0.05% ~ 0.10% |

建议同时测试三种成本情景：

1. 乐观成本；
2. 标准成本；
3. 悲观成本。

最终参数必须在标准成本和悲观成本下仍然有效。

# 十、防过拟合规则

## 1. 不选择孤点最优参数

如果某组参数收益很高，但参数轻微变化后收益大幅下降，不选。

应选择一片参数区域都表现较好的组合。

## 2. 不选择交易次数过少的参数

如果参数很苛刻，7 年只交易几十次，即使收益很高，也可能没有统计意义。

建议最低要求：

```text
2020—2026 总交易次数 >= 100
```

更理想是：

```text
2020—2026 总交易次数 >= 200
```

## 3. 不选择只靠单一年份赚钱的参数

如果某组参数主要收益来自 2020 或 2021，而其他年份表现平平甚至亏损，不适合作为实盘参数。

## 4. 做参数扰动测试

最终参数每个核心参数上下扰动 1—3 档，检查结果是否仍然较好。

例如：

```text
day2_min_pct_chg = -3.0
测试 -3.3、-3.2、-3.1、-3.0、-2.9、-2.8、-2.7
```

## 5. 做成本压力测试

最终参数至少要通过：

- 正常交易成本；
- 滑点扩大 2 倍；
- 印花税和佣金不利变化；
- 买入价上浮 0.1%—0.3%；
- 卖出价下浮 0.1%—0.3%。

# 十一、最终推荐技术路线

## 最优推荐路线

```text
候选事件表预生成
        ↓
NumPy / Numba 高速回测核心
        ↓
Optuna 贝叶斯优化搜索参数
        ↓
前 100 组参数局部网格精筛
        ↓
walk-forward 样本外验证
        ↓
参数扰动测试
        ↓
成本和滑点压力测试
        ↓
输出实盘推荐参数
```

## 为什么这样最快最好

1. 先生成候选事件表，避免每组参数重复扫全市场；
2. Numba 编译回测核心，比普通 pandas 循环快很多；
3. Optuna 比暴力枚举更快找到优质区域；
4. 局部网格精筛满足“0.1 或更低步长”的精细要求；
5. walk-forward 验证避免过拟合；
6. 扰动测试保证参数不是孤点最优。

# 十四、最终实操建议

## 1. 第一优先方案

优先采用：

```text
Optuna 贝叶斯优化 + Numba 高速回测 + 局部网格精筛
```

这是当前最适合本策略的方案。

## 2. GPU 使用建议

如果当前代码还没有向量化，先不要急着上 GPU。

正确顺序是：

1. 先把数据整理成候选事件表；
2. 再用 NumPy 数组化；
3. 再用 Numba 加速；
4. 如果仍然不够快，再考虑 CuPy 或 WSL2 + RAPIDS。

多数情况下，Numba + 多进程已经能达到很高速度。

## 3. 最终参数选择标准

最终不选择“收益最高”的参数，而选择：

```text
收益较高 + 回撤较低 + 每年稳定 + 交易次数足够 + 参数扰动不敏感
```

这是分歧流2策略从回测走向实盘最重要的原则。
