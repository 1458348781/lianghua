# 量化平台主体流程说明

本文档根据 `D:\lianghua` 目录下的代码、配置、数据、脚本、方案文档和运行产物整理，说明当前项目的主体流程、关键文件职责、数据流向和清理规则。

## 1. 项目总览

当前目录包含三条主要线索：

1. `quant-platform`：主体程序，一个本地可视化量化研究与盯盘平台。
2. `parameter_sweep.py`、`result*`：围绕“分歧战法/分歧流2”的参数枚举、随机优化和分钟级回测结果。
3. `deep_learing`：基于历史信号构建样本、训练 LightGBM 模型，并在实时/回测中辅助打分。

主体平台已经形成了比较完整的闭环：

```text
行情数据下载/导入
    -> SQLite 本地库
    -> 股票池/候选池筛选
    -> 策略信号生成
    -> 日线或分钟级回测
    -> 指标、净值、交易明细
    -> Web 页面展示
    -> 实时盯盘、邮件/微信提醒、持仓跟踪
```

## 5. 策略层流程

核心文件：

- `quant_platform/strategy.py`
- `strategy_plan.py`
- `方案/策略.docx`
- `方案/计算参数.docx`

`strategy.py` 中注册的策略由 `strategy_catalog()` 暴露给前端/API，`create_strategy(name, params)` 根据名称创建实例。

当前主要策略：

| 策略类 | 作用 |
|---|---|
| `MovingAverageStrategy` | 均线择时，短均线上穿/强于长均线时持仓 |
| `MomentumStrategy` | 动量月度调仓，按回看收益选 Top K |
| `DivergenceStrategy` | 分歧战法/分歧流2主体策略 |
| `GapTStrategy` | 隔日 T 思路，围绕支撑确认、乌云盖顶、放量长上影等规则 |
| `DivergenceFlowStrategy` | 更贴近方案文档的分歧流，先确认分歧，再等回调均线附近买入 |

### 5.1 分歧流2的核心逻辑

`DivergenceStrategy` 是当前优化、回测和实时候选池中最核心的策略。整体按 T、T+1、T+2 三段判断：

1. T 日：必须出现有效涨停，人气和一致性成立。
2. T 日过滤：排除 ST、一字板、价格过低/过高、换手率异常、量比异常、前期振幅/涨幅不足等情况。
3. T+1：要求进入“可控分歧”，涨跌幅、量能、收盘位置、上影线、收盘价相对 T 日收盘价都要在参数范围内。
4. T+2：要求高开幅度在指定区间，盘中从开盘价继续上冲，并突破 T+1 高点。
5. 买入价：通常取 `max(T+1 高点, T+2 开盘价 * 上冲阈值)` 附近。
6. 卖出：按止损、走强止盈、持仓天数到期等规则退出。

`方案/计算参数.docx` 中记录的一组典型参数为：

| 参数 | 含义 |
|---|---|
| `max_positions = 3` | 最大持仓数 |
| `hold_days = 3` | 持仓天数 |
| `stop_loss = -0.03` | 止损 -3% |
| `strong_close_pct_chg = 5` | 走强阈值 5% |
| `min_turnover = 2` | T 日换手率下限 |
| `max_turnover = 25` | T 日换手率上限 |
| `day1_min_volume_ratio = 1` | T 日成交量不低于过去 5 日均量 |
| `day1_max_volume_ratio = 4` | T 日成交量不高于过去 5 日均量 4 倍 |
| `range_min_amplitude_30 = 0.18` | 过去 30 日振幅不低于 18% |
| `day2_min_pct_chg = -3` | T+1 涨跌幅下限 |
| `day2_max_pct_chg = 8` | T+1 涨跌幅上限 |
| `entry_min_open_gap_pct_chg = 1` | T+2 高开下限 |
| `entry_max_open_gap_pct_chg = 5` | T+2 高开上限 |
| `entry_min_high_from_open_pct_chg = 4` | T+2 盘中最高价相对开盘至少涨 4% |

## 6. 回测层流程

核心文件：

- `quant_platform/backtest.py`
- `quant_platform/minute_backtest.py`
- `quant_platform/metrics.py`
- `scripts/run_backtest.py`

### 6.1 日线回测

`BacktestEngine` 的核心流程：

1. 根据 `symbols/start_date/end_date` 从 `DataPortal` 取交易日历和行情。
2. 创建 `Context`，把组合、历史行情和策略配置传给策略。
3. 每个交易日先处理盘前逻辑 `before_open()`。
4. 执行上一交易日留下的 T+1 开盘订单。
5. 调用策略 `on_bar()` 生成新的目标仓位订单。
6. 根据 `execute_now` 判断当日收盘执行或下一交易日开盘执行。
7. 每日记录净值、现金、市值、持仓。
8. 最后计算收益、回撤、波动率、夏普、胜率、交易次数等指标。

交易假设：

- 使用日线前复权价格。
- 默认 T 日收盘后生成信号，T+1 开盘成交。
- 未完整处理涨跌停、停牌、幸存者偏差和分红现金流。
- A 股按 100 股整数手成交。
- 手续费、滑点、印花税按固定比例近似。

### 6.2 分钟级回测

`minute_backtest.py` 和 `scripts/optimize_divergence2_minute.py` 用于更贴近盘中触发的验证：

1. 先用日线找出 T/T+1 形态合格候选。
2. 查询 T+2 当日分钟线。
3. 在分钟线上寻找突破 T+1 高点、开盘后涨幅达到阈值等触发点。
4. 根据分钟线处理盘中止损/出场。
5. 输出组合级和年度级指标。

## 7. 参数优化流程

相关文件：

- `parameter_sweep.py`

### 7.1 顶层 `parameter_sweep.py`

这是早期/独立版参数扫描脚本，直接引用 `quant-platform` 包：

```text
parameter_sweep.py
    -> MarketDatabase
    -> create_strategy("divergence_tactic")
    -> BacktestEngine
    -> result/*.csv + result/*.json
```

它把不同年份数据预加载到内存，再用多进程跑参数组合，最后聚合各年表现。

### 7.2 `scripts/optimize_divergence2_params.py`

这是平台目录内更完整的随机参数优化器，支持：

- 全局随机采样
- 围绕历史较优参数做局部采样
- 多年份分别回测
- 汇总年度表现
- 按收益、回撤、交易数等综合打分

### 7.3 `scripts/auto_optimize_divergence2.py`

这是自动分阶段调参编排器：

```text
smoke_100
    -> direction_500_a / direction_500_b
    -> formal_2000
    -> formal_3000 / formal_3000_w4
    -> auto_best_overall.json
```

它会读取每阶段最优结果，根据质量决定是否继续下一阶段。

### 7.4 `result*` 目录含义

| 目录 | 内容 |
|---|---|
| `result` | 早期 `parameter_sweep.py`、参数枚举和旧参数复测结果 |
| `result2` | 分歧流2日线随机优化、自动分阶段优化结果 |
| `result3` | 分钟级参数优化、种子扫描、最佳参数精修结果 |

这些结果可用于追溯参数来源，但不影响平台启动。

### 可清理

| 类型 | 处理建议 |
|---|---|
| `__pycache__`、`*.pyc` | Python 编译缓存，可直接删除 |
| 与现有文件哈希完全一致的备份 | 可删除，例如 `parameter_sweep_legacy_backup_20260510.py` 与 `parameter_sweep.py` 完全一致 |

| 旧的大量 runtime 日志 | 可按日期归档或删除 |
| `result*` 中 smoke/inmemory/失败空结果 | 若不再复盘，可删除 |
| `.venv` | 可再生成，但当前缺少完整依赖说明，暂不建议直接删除 |
| `data/cache/limit_up_pullback_baostock` | 旧 CSV 缓存，SQLite 已有主库时可删除，但若要复现实验则保留 |
